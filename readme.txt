# Introdução

Após realizar um estudo sobre os usuários, seus propósitos e necessidades e criar seus
protótipos podemos iniciar a criação de um protótipo em HTML para o nosso projeto.
Nas últimas aulas, falamos sobre a necessidade de que um projeto seja adequado tanto
para PC quanto para dispositivos móveis. Por isso, criar um projeto Responsivo e MobileFirst.

# Objetivo

Neste trabalho, o objetivo é desenvolver e transformar os protótipos criados para o
trabalho anterior em interfaces gráficas responsivas utilizando HTML, CSS e JavaScript.
As interfaces gráficas devem ser Mobile First e atender aos seguintes dispositivos:

- 320px — 480px: Mobile devices
- 769px — 1024px: Small screens, laptops

# Enunciado inicial do trabalho

Uma empresa desenvolve produtos para intervenções cardiovasculares, como
cateteres. As vendas são realizadas para governos, hospitais, clínicas e médicos. Hoje,
um dos principais desafios da empresa é o pós-venda. O pós-venda é a etapa do
processo de venda que vem depois da compra. O objetivo do pós-venda é reter o cliente
e gerar novos ganhos para a empresa. Dessa forma, a jornada do consumidor continua
depois de concretizar a compra. Após a compra, toda informação de interação com o
produto pode direcionar melhorias que aumentem a satisfação do cliente. Assim, a
empresa deseja desenvolver um aplicativo para dispositivos móveis que apoie o pósvenda. Em equipe, empregue a metodologia do diamante duplo do design, para
desenvolvimento do aplicativo.
